About VisualQA

Suggest a tool GitHub

Building an open educational resource for neuroimaging quality assurance.

Quality assurance is an important part of neuroimaging data processing, but it is also one of the least standardized processes. Two people can look at the same scan and disagree about whether it passes — often because nobody ever showed them what a given artifact actually looks like.

VisualQA aims to close that gap with annotated examples of common imaging artifacts, practical visual inspection guidelines, and curated resources — so researchers can learn QA, and perform it more consistently.

The project began around MRI and fMRI. It is now expanding to EEG, with QA resources planned for both. If you work in either area — or care about building open educational resources — you're welcome to join in.

How health status works. The curated tools in the directory are checked weekly by a GitHub Actions cron job against each tool's repository.

  • Active — committed within 6 months
  • Quiet — 6 to 24 months
  • Inactive — 2+ years or archived
  • Not on GitHub

What gets included. Resources need to help someone actually do QA — spot an artifact, judge a scan, or automate a check — and be open and actively maintained. Small or little-known tools are welcome as long as they solve a real problem and are being worked on.

How to contribute. Open an issue directly on GitHub, or submit a pull request. Each tool is a YAML entry: name, url, description, language, modality (top-level: MRI, EEG), optionally submodality (e.g. fMRI, diffusion) and github (owner/repo for health tracking), and added_by (with optional added_by_github handle) to credit whoever suggested it.
